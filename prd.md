### 제품 방향

- **최소 기능(v1)으로 시작 → 반응 확인 → 단계적으로 확장**
- 확장 순서(권장): **내 착장 CRUD → 의류 구분 입력 → 외부 레퍼런스 저장 → AI 추천**

---

## v1 (1단계) – 스타일로그 CRUD (MVP)

**목표:** “기록이 쌓이고, 다시 꺼내볼 수 있다”만 성립하면 성공

### 범위(In Scope)

- **Create(작성)**
    - 사진 업로드: 1~3장
    - 날짜(기본 오늘, 수정 가능)
    - 메모(선택)
    - 즐겨찾기(선택)
    - 태그(선택: 체감/상황 등) — *없어도 됨(최소화 가능)*
    - (옵션) 위치/날씨 자동 입력 — *v1.1로 미뤄도 OK*
- **Read(조회)**
    - 리스트(최신순)
    - 상세(사진 1~3, 날짜, 메모, 즐겨찾기)
- **Update(수정)**
    - 사진/날짜/메모/즐겨찾기 수정
- **Delete(삭제)**

### 비범위(Out of Scope)

- 의류 구분(상의/하의/아우터) 입력
- 외부 레퍼런스 저장
- AI 추천
- 커뮤니티/공유

### 화면(최소 IA)

- 홈(리스트)
- 작성/수정 폼
- 상세
- (선택) 필터/검색은 v1에서 생략 가능(혹은 즐겨찾기만 필터)

### 성공 기준(MVP)

- 사용자가 **일주일에 3회 이상 기록**할 수 있다
- “내 기록을 다시 열어봤다”가 발생한다(리스트/상세 재조회)

---

## v2 (2단계) – 의류 구분 입력(텍스트) + 필터 확장

**목표:** “내 옷 기준으로” 더 정밀한 회고/검색

### 추가 기능

- 로그에 의류 필드 추가
    - 상의 / 하의 / 아우터 / 신발(선택)
- 입력 UX
    - 텍스트 입력
    - 기존 입력값 자동완성(드롭다운)
- 필터 확장
    - 기온(있다면) + 의류 조합으로 필터
    - 예: “10도 & 상의=맨투맨”

---

## v3 (3단계) – 외부 레퍼런스 저장

**목표:** “내 착장”만이 아니라, “보고 저장한 레퍼런스”도 함께 코디 참고 자산으로 축적

### 모델 설계 제안

- 동일한 Log 구조에 `type` 추가:
    - `type = my_outfit` (내 착장)
    - `type = reference` (외부 레퍼런스)

### 레퍼런스 기능

- 이미지 업로드(1~3장) 또는 링크/출처(선택)
- 태그/메모(선택)
- 리스트에서 타입 필터(내 착장만 / 레퍼런스만)

---

## v4 (4단계+) – AI 추천

**목표:** 축적된 데이터를 “추천”으로 전환

### 추천 범주

- 내부 데이터 기반 추천
    - 기온/체감/상황/의류 태그 기반 “비슷한 날의 성공 코디”
- 외부 데이터 기반 추천
    - 트렌드/룩북/커뮤니티/상품 데이터 결합(후순위)
